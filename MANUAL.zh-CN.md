# 一键生成亚马逊主图工作台使用手册

## 一、产品简介

**一键生成亚马逊主图工作台**是一款面向亚马逊卖家的 AI 内容生产工具。你只需要输入产品名称和产品描述，系统即可自动生成 5 条贴合产品的中文卖点文案，并为每条卖点生成对应的电商配图，最终支持一键导出 ZIP 包。

核心流程：**输入产品信息 → 生成卖点文案 → 生成主图素材 → 导出交付包**

## 二、核心功能

| 功能 | 说明 |
|------|------|
| **主图文案生成** | 基于 Gemini 模型，从产品描述中提炼出 5 条包含场景、人群和使用方法的卖点文案 |
| **配图生成** | 为每条卖点生成英文图像提示词，并调用 AI 图像模型产出电商级主图素材 |
| **参考图关联** | 支持上传白底产品图，生成图片时会参考产品外观，提升相关性 |
| **画布式工作台** | 单页画布布局，输入、文案、输出同时可见，适合连续编辑与复用 |
| **文案实时编辑** | 可手动修改卖点标题和正文，保存后重新生成对应图片 |
| **一键导出 ZIP** | 导出包含全部图片 + 元数据 JSON 的压缩包，方便上架使用 |

## 三、使用步骤

### 1. 创建项目

- 在首页点击「创建工作区」
- 填写「项目标题」、「产品名称」、「产品描述」
- 产品描述越详细，生成的卖点越贴合

### 2. 上传参考图（可选但推荐）

- 在画布左侧的输入节点中点击「上传」
- 建议上传一张白底产品图
- 这会让后续生成的配图更贴合你的产品外观

### 3. 生成卖点文案

- 点击「生成文案」：在现有输入基础上生成 5 条卖点
- 或点击「替换全部文案」：强制覆盖已有卖点，重新生成
- 生成过程中会显示进度条和阶段提示

### 4. 查看与编辑卖点

- 中间区域展示 5 条卖点节点
- 可修改「标题」和「正文」
- 点击「显示提示词」可查看该卖点对应的英文图像提示词
- 修改后点击「保存文案」

### 5. 生成图片

- 点击顶部工具栏的「生成全部图片」，为所有卖点批量生成配图
- 或在单个图片节点点击「重新生成图片」，单独刷新某张图
- 生成完成后，右侧区域显示成品图

### 6. 放大查看与导出

- 点击图片可放大查看
- 确认无误后，点击顶部「导出 ZIP」下载完整素材包

## 四、界面说明

```
┌─────────────────────────────────────────────────────────────┐
│ 返回首页   项目名称      适配画布   生成全部图片   导出 ZIP   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐      ┌─────────────┐      ┌─────────────┐ │
│   │   输入节点   │─────→│  卖点节点   │─────→│   输出节点   │ │
│   │  产品简介   │      │  5 条卖点   │      │   5 张图片   │ │
│   │  参考图片   │      │   可编辑    │      │   可重生图   │ │
│   └─────────────┘      └─────────────┘      └─────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

- **蓝色状态**：表示节点就绪或正常
- **黄色状态**：表示内容已编辑或图片待刷新
- **绿色状态**：表示图片已生成完成

## 五、注意事项

1. **API Key 配置**：首次使用需在 `.env.local` 中配置 `GEMINI_API_KEY`，文案和图片生成均依赖 Gemini API。
2. **网络环境**：生成文案和图片需要能够访问 Gemini 服务，请确保网络畅通。
3. **产品描述质量**：描述中包含材质、功能、使用场景、目标人群等信息，能显著提升卖点质量。
4. **已有数据生效范围**：修改 prompt 规则或布局逻辑后，只有**新生成**的卖点会生效；旧卖点需点击「替换全部文案」或「重新生成图片」才能更新。

## 六、快速启动

```bash
# 安装依赖
npm install

# 配置环境变量
cp .env.example .env.local
# 编辑 .env.local，填入 GEMINI_API_KEY

# 初始化数据库
npx prisma generate
npx prisma migrate dev --name init

# 启动开发服务器
npm run dev
```
